┌─ CRITICAL ARCHITECTURE (v1.4.x — PURE P2P, ZERO SERVEUR, ZERO LOCALHOST) ──┐
│                                                                             │
│ TRANSPORT STACK:                                                            │
│  1. Trystero (@trystero/torrent) → signaling via trackers BitTorrent publics│
│     Trackers: wss://tracker.openwebtorrent.com                             │
│               wss://tracker.btorrent.xyz                                   │
│               wss://tracker.fastcast.nz                                    │
│               wss://tracker.webtorrent.dev                                 │
│  2. WebRTC DataChannels → transport réel (messages, réactions, profils)    │
│  3. GunDB radisk → persistance locale (AppData/Roaming/Mesh/radata/)       │
│  4. localStore.ts → AppData/Roaming/Mesh/mesh-data/ (JSON files, auth)     │
│                                                                             │
│ AUTH (100% LOCAL, aucun serveur):                                           │
│  - users.json dans AppData: {username, passwordHash(bcrypt), createdAt}    │
│  - userIndex GunDB: annuaire public {exists: true, username, createdAt}    │
│    → uniquement pour valider si un pseudo existe cross-machine              │
│    → AUCUN mot de passe dans userIndex                                      │
│  - Session: localStorage mesh_session_user                                 │
│                                                                             │
│ FICHIERS LOCAUX (AppData/Roaming/Mesh/mesh-data/):                         │
│  - users.json        → comptes locaux                                      │
│  - userServers.json  → {[username]: serverId[]}                            │
│  - servers.json      → {[serverId]: Server}                                │
│  - channels.json     → {[serverId]: Channel[]}                             │
│  - members.json      → {[serverId]: Member[]}                              │
│                                                                             │
│ MESH.TS (src/mesh.ts):                                                      │
│  - joinMeshRoom(roomId, myProfile?) → Trystero room                        │
│  - onPeerJoin: sendProfile(myProfile, [peerId]) via makeAction('profile')  │
│  - broadcastProfile(myProfile) → update toutes les rooms actives           │
│  - peerProfiles: Record<string, any> → profils des pairs connectés         │
│  - leaveAllMeshRooms() → appelé au logout                                  │
│                                                                             │
│ FRONTEND:                                                                   │
│  - React app dans Electron BrowserWindow (frameless, custom title bar)     │
│  - Builds via: npm run build → tsc -b + vite build → dist/                 │
│  - Vite config uses manualChunks as FUNCTION (not object) for Rolldown     │

┌─ RÈGLE ARCHITECTURALE CRITIQUE : GUNDB ≠ PERSISTANCE ──────────────────────┐
│                                                                             │
│ PRINCIPE FONDAMENTAL (v1.3.x+) :                                           │
│                                                                             │
│   GunDB  →  CACHE MESSAGES UNIQUEMENT (radisk local, peers: [])            │
│   localStore (IPC) → TOUTE la persistance (profils, serveurs, membres...)  │
│   Trystero (WebRTC) → TRANSPORT TEMPS RÉEL (profils, présence, réactions)  │
│                                                                             │
│ RÈGLES STRICTES :                                                           │
│                                                                             │
│  ✗ INTERDIT — GunDB pour données persistantes :                            │
│    - Profils (useProfile.ts) → localStore profiles.json                    │
│    - Serveurs/métadonnées (useServers.ts) → localStore servers.json        │
│    - Membres (useMembers.ts) → localStore members.json                     │
│    - Channels (useChannels.ts) → localStore channels.json                  │
│    - Rôles (useRoles.ts) → localStore roles.json                           │
│    - DMs (useDMs.ts) → localStore dms.json                                 │
│    - Bots config (useBotEngine.ts) → localStore bots.json                 │
│    - Bannières/tags serveur → localStore servers.json via updateServerFull │
│    - AutoMod config → localStore automod.json                              │
│    - Webhooks → localStore webhooks.json                                   │
│    - Rôles canaux / perms → localStore channel_perms.json                  │
│                                                                             │
│  ✓ AUTORISÉ — GunDB limité à :                                             │
│    - gun.get('messages') → cache local radisk pour l'historique chat       │
│    - gun.get('reactions') → cache local radisk                             │
│    - gun.get('userIndex') → annuaire public cross-machine (pseudo seul)    │
│    - gun.get('servers').get(id) → découverte cross-machine pour joinServer  │
│    - gun.get('invites') → codes d'invitation cross-machine                 │
│                                                                             │
│  ✓ TRANSPORT TEMPS RÉEL (Trystero uniquement) :                            │
│    - Profils en ligne → broadcastProfile() / onPeerProfile()               │
│    - Présence membres → joinMeshRoom('presence_{serverId}')                │
│    - Mises à jour serveur → joinMeshRoom('server_meta_{id}')               │
│    - Messages → makeAction('message') dans useSocket.ts                    │
│    - Réactions/typing → makeAction('reaction') / makeAction('typing')      │
│                                                                             │
│ PATTERN STANDARD localStore :                                               │
│   // Lire                                                                   │
│   const data = await readLocal<MyType>('file.json') || {}                  │
│   // Écrire                                                                 │
│   await writeLocal('file.json', { ...data, [key]: value })                 │
│                                                                             │
│ POURQUOI :                                                                  │
│   GunDB avec peers:[] ne se synchronise PAS cross-machine pour les données  │
│   persistantes → tout ce qui est écrit dans GunDB hors messages est PERDU  │
│   au redémarrage ou sur une autre machine. localStore = AppData/JSON =      │
│   seule source de vérité fiable sur la machine locale.                     │
│                                                                             │
│ MIGRATION : Quand tu modifies un hook, remplace :                          │
│   gun.get('X').get(id).put(data)   →   writeLocal('X.json', data)         │
│   gun.get('X').get(id).on(cb)      →   readLocal('X.json') au montage     │
│   gun.get('X').get(id).once(cb)    →   readLocal('X.json') au montage     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
